Technical Details

Root Cause Analysis

The issue was caused by aggressive FOUC (Flash of Unstyled Content) prevention CSS that blocked all pointer events on portfolio elements until the entrance animation completed. The CSS rule:

body:not(.entrance-complete) .portfolio-container { pointer-events: none; }

This made all interactive elements unclickable during the entrance animation period.

Solution

Replaced pointer-events: none with visibility: hidden for visual hiding, and added explicit pointer-events: auto when the entrance is complete.

Fallback Mechanisms

  • 5-second timeout: Automatically enables portfolio if entrance animation fails
  • Error handling: Catches entrance animation errors and enables portfolio immediately
  • Manual override: Red button for immediate testing (remove in production)
